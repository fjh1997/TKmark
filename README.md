# 超星作业批改脚本

这个仓库用于超星学习通作业批改、考勤/考试数据采集和课程最终成绩统计。

## 功能

- 通过当前已登录的浏览器 CDP 会话读取超星考试成绩。
- 读取超星讨论区里“存储 XSS 实验中成功弹窗”的课堂成功名单。
- 读取课堂考勤系统的学期统计数据。
- 合并本地作业评分结果，生成三个班的最终成绩 HTML、XLSX 和 JSON。
- 自动搜索平时成绩/期末成绩比例，使及格人数尽可能多。

## 安全说明

仓库不保存 cookie、token、API key、学生成绩明细或浏览器用户数据。

本地输出目录 `out/`、采集缓存目录 `data/`、`.env` 和 `my-user-data/` 已加入 `.gitignore`。运行脚本前请确认浏览器已登录对应系统，脚本只通过当前浏览器会话临时读取数据。

## 环境

- Node.js 22 或更高版本。
- 已打开远程调试端口的 Edge/Chrome。
- WSL 场景可用 `wsl-host-cdp` 桥接宿主机浏览器。

## 生成最终成绩

默认读取相邻目录里的当前作业排名文件：

```bash
npm run final-grades -- \
  --cdp http://127.0.0.1:59224 \
  --assignment-json ../chaoxing-grader/score_rankings_behinder_adjusted.json \
  --alias-json data/discussion-aliases.json \
  --excluded-json data/excluded-students.json \
  --out-dir out/final-grades-20260706
```

输出：

- `final-grade-report.html`
- `final-grade-report.xlsx`
- `final-grade-report.json`

## 生成考勤点名册

可以把最终成绩报告里的考勤数据整理成打印版点名册。脚本会按成绩录入模板中的学生顺序生成三个班的工作表，每 25 人一页，样式接近纸质“学生点名册”。

```bash
python3 scripts/build_attendance_roll_call_xlsx.py \
  --report-json out/final-grades-20260706/final-grade-report.json \
  --template-dir ../期末成绩录入模板 \
  --course-name 信息安全代码审计 \
  --out out/attendance-roll-call/三班考勤点名册.xlsx
```

标记规则：

```text
到课/  缺课O  病假⊙  事假⊕  公假公  未填原因请假假  迟到◎  早退X
```

如果 `final-grade-report.json` 中包含 `attendanceEvents` 逐次考勤明细，脚本优先按逐次事件填写每个日期的状态。如果只有汇总字段和 `leaveReasons`，则只能把有日期的请假填入对应日期，缺课、迟到、早退等无日期汇总会写入备注列。

## 评分合成规则

默认先计算平时原始分：

```text
平时原始分 = 作业均分 * 85% + 考勤分 * 10% + 课堂讨论成功弹窗加分 5 分
```

平时原始分封顶 100 分。默认再按同班最高原始分归一化：

```text
平时成绩 = 平时原始分 / 同班最高平时原始分 * 100
```

这样每个班平时表现最高的学生为 100 分，其他学生按相对表现等比例赋分，不改变班内平时排名。需要关闭时可用：

```bash
node scripts/final-grade-report.mjs --ordinary-normalization none
```

考勤分默认从 100 分开始扣：

```text
未签到 * 3 + 普通请假 * 1.5 + 公假/因公请假 * 0.5 + 迟到 * 1 + 早退 * 1
```

考勤会先读取学生学期统计，再逐条读取教师端考勤详情里的学生状态。请假类型来自详情接口的 `leaveReason`，例如“事假（公）”“事假（私）”“病假”。默认扣分：

```text
未签到 * 3 + 公假 * 0.5 + 病假 * 1 + 私假 * 1.5 + 未分类请假 * 1.5 + 迟到 * 1 + 早退 * 1
```

如果详情里请假原因为空，脚本归为“未分类请假”。

最终成绩比例默认在“平时 20%-60%”的常规范围内搜索，目标是及格人数最多；如果及格人数相同，选择最接近平时 40% / 期末 60% 的比例。报告中也会同时列出 0%-100% 无限制搜索的结果，便于复核。

## 分班归一化最终成绩策略

最终成绩允许三个班分别选择平时成绩归一化方法和平时/期末比例。搜索时不使用固定最低分地板，而是在每个班内部保持平时原始分排名顺序，再用幂函数压缩低分段：

```text
平时成绩 = 100 * (平时原始分 / 本班最高平时原始分) ^ gamma
最终成绩 = 平时成绩 * 平时比例 + 期末成绩 * 期末比例
```

分班搜索约束：

- 平时比例限定在 20%-60%，期末比例为剩余比例。
- `gamma` 在候选区间内逐步搜索，`gamma` 越小，低平时分学生被抬升越多；`gamma = 1` 等价于线性归一化。
- 优先选择全员及格的方案。
- 多个方案都能全员及格时，优先选择最终成绩标准差更大的方案，保证班内区分度。
- 标准差接近时，选择平均分更低且平时比例更接近 40% 的方案，避免无必要的整体抬分。

当前采用的分班参数：

```text
信安24-01：    平时成绩 = 100 * (平时原始分 / 本班最高平时原始分) ^ 0.53，平时 20%，期末 80%
信安24实验班：平时成绩 = 100 * (平时原始分 / 本班最高平时原始分) ^ 0.23，平时 60%，期末 40%
信安2504：     平时成绩 = 100 * (平时原始分 / 本班最高平时原始分) ^ 0.10，平时 34%，期末 66%
```

这些参数的目标是：在不直接给固定底分的前提下，使每个班全员及格，同时保留尽可能高的最终成绩区分度。实际导出 Excel 时，应把每个班的方案摘要、候选方案、成绩表分开保存，便于人工复核。

## 期末卷面修正规则

期末卷面以超星考试成绩为基础。若发现客观填空题因格式、大小写、空格、下划线或轻微重复字符导致自动判分失败，可以按可审计规则补分：

- 只处理当前未满分的客观填空题。
- 学生答案必须包含 `flag{...}` 格式。
- 该 flag 与同题文本中出现的参考 flag 标准化后相同，或相似度足够高。
- 标准化会忽略大小写、空格、下划线、冒号等非字母数字字符。
- 空题、无 flag 格式、或明显不是本题 flag 的答案不补分。
- 补分只补该客观填空题满分，不额外补主观题。

主观题补分规则单独处理：已作答的主观小题可以根据阅卷标准重评，空白题仍为 0 分。最终报告应保留原期末分、补分值和修正后期末分，方便追溯。

## 常用参数

```bash
node scripts/final-grade-report.mjs --help
```

可调参数包括 CDP 地址、作业 JSON 路径、输出目录、考勤学期、教师编号、平时成绩比例搜索范围和搜索步长。

如果讨论区成功名单使用拼音、缩写或昵称，本地创建 `data/discussion-aliases.json`：

```json
{
  "nickname_or_pinyin": "真实姓名"
}
```

`data/` 已被忽略，不会推送到 GitHub。

如果 roster 里混入教师、休学或不应统计的人员，本地创建 `data/excluded-students.json`：

```json
{
  "excluded": [
    { "studentNo": "学号或工号" },
    { "name": "姓名" }
  ]
}
```

排除名单只影响最终成绩统计、比例优化和导出的 Excel，不会修改原始采集数据。
